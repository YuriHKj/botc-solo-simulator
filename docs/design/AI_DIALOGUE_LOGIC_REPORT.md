# AI 对话实现调研报告（2026-04-24）

## 一、当前实现总览

AI 对话是**本地规则引擎**，不依赖在线 LLM。
主要链路文件：
- `scripts/ai.js`：意图识别、信念更新、私聊/公聊文本生成
- `scripts/app.js`：对白天流程与 UI 事件进行调度
- `scripts/ui.js`：提问入口、快捷问法、私聊弹窗、对话舞台展示
- `scripts/engine.js`：阶段限制、私聊次数、邪恶方首夜互认等规则约束

结论：目前不是“随机胡说”，而是基于可解释规则在运行。

## 二、私聊链路（你和电脑玩家）

入口函数：`runPrivateWhisper(...)`（`scripts/ai.js`）

执行顺序：
1. 阶段校验：必须是白天 `private` 阶段。
2. 目标校验：目标必须是存活的非人类玩家。
3. 次数扣减：`consumePrivateChat(...)`。
4. 刷新推理：`refreshAIBeliefs(...)`。
5. 意图识别：`detectIntent(...)`（关键词+规则评分）。
6. 交互信号回写：`applyPrivateChatSignals(...)`，把你“施压/维护”倾向写回 AI 偏置。
7. 回答生成：
   - 邪恶同阵营：`composeEvilAllianceResponse(...)`
   - 普通私聊：`composePrivateResponse(...)`
8. 写入日志与时间线：`addLog(...)` + `pushTimeline(...)`。

## 三、公聊链路（AI 互相发言）

入口函数：`runAIDiscussion(...)`（`scripts/ai.js`）

执行顺序：
1. 刷新 AI 信念。
2. 按人格与局势决定是否公开报身份。
3. `composePublicLine(...)` 生成每位 AI 的一条发言。
4. 写入 `speech` 日志与时间线。

## 四、玩家输入 prompt 的判定机制

当前机制：**规则判定 + 关键词评分**，不是神经网络语义理解。

- 主要意图：`suspect/reason/trust/claim/vote/night/compare/plan/generic`
- 识别位置：`detectIntent(...)`
- 提及玩家提取：支持名字与“x号”格式

优点：
- 离线可运行、稳定、可控、可复盘

不足：
- 复杂否定句、多意图句、隐喻表达会误判
- 同一句里“主问题+补充问题”时容易偏向单一意图

## 五、这次已落地的人机交互优化（第一轮）

### 1) 提问入口增加“意图预览”
已在两处加入：
- 左侧私聊区：`#whisperIntentPreview`
- 私聊弹窗：`#chatDramaIntentPreview`

作用：你在点击发送前，先看到系统将按哪类问题处理（如“怀疑判断/投票提名/身份表态”）。

### 2) 快捷问法接入意图提示
快捷按钮现在带有 `intentHint`，会传入 AI 引擎，减少“按钮文案已很明确却被误判”的情况。

### 3) 输入判定改进
`detectIntent(...)` 增加了：
- 外部 `intentHint` 融合
- 否定语句修正（如“不是在怀疑他”不再强判为怀疑意图）

### 4) 私聊回复更“人话”
`composePrivateResponse(...)` 增加“量化开关”：
- 你问概率/百分比时，AI 给数字
- 普通问法默认给定性判断，减少机械百分比刷屏

## 六、目前仍存在的交互问题

1. 私聊入口仍是“双通道”（左侧栏 + 私聊弹窗继续追问），对新手可能有一点学习成本。
2. 公聊目前是“批量结算 + 时间线回放”，戏剧性节奏还可增强（逐句推进、可中断）。
3. 当前 AI 仍是“可解释规则智能”，不是语义大模型，对超自由文本的理解上限明确。

## 七、下一步优化建议（建议按顺序）

1. **入口收敛**：把“发起私聊”统一为魔典点人 + 弹窗，左侧保留流程按钮与摘要，不再承载完整输入。
2. **公聊舞台化**：一轮公聊改为逐句动画推进（带发言顺序与短暂停顿），提升辩论沉浸感。
3. **多意图问句拆分**：对“你怀疑谁+你会不会投”这类句子做双意图拆解，按主次回答。
4. **回答一致性约束**：为每个 AI 增加“本日口径锁”，减少同日自相矛盾。
