# Unity Next Steps Roadmap

日期：2026-05-07

本文用于梳理 BOTC Solo Simulator 接下来怎么推进。当前方向保持不变：JS Core 是唯一规则、AI、权限和结算引擎；Unity 只做更好的 UI、动画和交互承载。

## 当前状态

Unity demo 已经从静态视觉原型进入“可运行 demo”阶段：

- `unity_viewmodel.json`：由 JS Core 导出，Unity 读取。
- `unity_action.json`：Unity 写入玩家行动，`unity_action_bridge.mjs` 转交 JS Core。
- 已覆盖主要权限边界：非恶魔主视角看不到真实恶魔伪装，非全知视角不泄露非主视角真实身份。
- 已迁移主要 UI 面板：魔典 token、顶部 HUD、底部托盘、私聊面板、资料抽屉、投票仪式面板、动态行动表单。
- 已有验证链路：`npm test`、JS 语法检查、Unity build、Unity exe smoke。

当前主要短板不是“能不能跑”，而是“正式 UI 的信息架构、操作密度和视觉节奏还不稳定”。

## 优先级原则

1. 先让 Unity demo 更清楚、更不挤、更像一个完整可玩界面。
2. 保持 JS Core 规则驱动，不在 Unity 里补规则。
3. 每迁移或重做一个交互块，都保留契约测试和 Unity smoke。
4. AI 对话质量和规则边界要记录，但暂时不抢 UI demo 的优先级。

## P0：下一轮最该做的 UI 收束

### 1. 底部动作区改为“常用动作 + 更多动作抽屉”

现状：

- 已完成第一版：底部从 16 个常驻按钮收束为 8 个常用入口。
- 低频动作已移入右下 `更多动作` 抽屉。

已落地：

- 底部保留：私聊、公聊、提名、投票、行动、资料、更多、收起。
- `更多动作` 包含：询身、骗身、保密、编夜信、夜间、白天、说书人、标记、剧本、复盘、全知。
- `行动` 会按当前 viewmodel 优先打开 Storyteller、夜间或白天行动表单。

后续可继续优化：

- 按阶段禁用或弱化不可用动作，而不是永远全亮。
- 给“更多”按钮增加当前抽屉开启态高亮。

验收：

- 底部不再出现 4x4 密集按钮阵列。已完成。
- 主视角玩家一眼能看懂“现在最该点什么”。第一版完成。
- 不改变 `unity_action.json` action type。已完成。

### 2. Token 详情从底部挪到独立轻量面板

现状：

- 已完成第一版：点击 token 后详情写入独立 `Token Inspector`。
- 底部 dialogue 只保留短提示，不再承担玩家详情卡。

已落地：

- `Token Inspector` 位于左下，展示身份显示、状态、怀疑值、魔典标记、认知角色和提醒物。
- Inspector 内置 `私聊`、`提名`、`关闭` 快捷按钮。
- `Esc` 可关闭 Inspector。
- 内容只使用 `players[]` 中已由 JS Core/viewmodel 过滤过的可见字段。

验收：

- 底部 dialogue 不再承担玩家详情卡。已完成。
- 切换 token 时详情即时刷新。已完成。
- 不泄露未授权身份。沿用 viewmodel 权限契约。

### 3. 私聊面板视觉定稿

现状：

- 已完成第一版视觉定稿：面板从纵向堆叠改为正式双栏弹窗。
- 私聊过程不再依赖底部长文本说明。

已落地：

- 左侧：目标和最近私聊历史。
- 右侧：身份声称、夜间信息输入、保密开关和发送状态。
- 底部：`询问身份`、`发送私聊`、`关闭` 三个主操作。
- 发送后在面板内显示“等待 JS Core 刷新回复”的状态反馈。

验收：

- 私聊过程不再依赖底部长文本说明。已完成。
- 最近私聊历史清楚且不会遮挡魔典。第一版完成。
- 发送后 AI 回复仍来自 timeline/viewmodel。已保持。

### 4. 投票动画升级为 token 式仪式镜头

现状：

- 已完成第一版 token 式仪式镜头。

已落地：

- 投票面板使用 `voteCeremony.voters[]` 生成环形 token 镜头。
- 当前被询问者高亮；赞成显示金色 `举`，反对显示 `落`，弃权显示 `弃`。
- 顶部保留提名链、实时计票和最终 `resultText`。

验收：

- `voteCeremony.voters[]` 仍是唯一数据来源。已完成。
- 动画结束结果与 JS Core `resultText` 一致。已完成。
- 行列表不再是唯一视觉主体。已完成。

## P1：补齐可玩 demo 闭环

### 1. 夜间 / 白天复杂行动表单

现状：

- 已完成第一版复杂行动表单。

已落地：

- `actionForms[]` 的 input type 已映射到对应控件：
  - player targets：token 多选。
  - role guess：角色选择器。
  - mode：分段按钮。
  - text/number：输入框。
- guess 类输入支持选择玩家 + 身份后提交给 JS Core。
- 表单会展示最小/最大选择数、当前选择和确认状态。

验收：

- Fortune Teller 这类多目标行动不用再依赖自动合法选择。已完成。
- 复杂角色至少有可解释的通用输入方式。第一版完成。

### 2. Storyteller 队列面板

现状：

- 已完成独立 Storyteller 队列面板。

已落地：

- 独立面板展示 `storytellerQueue[]`。
- 展示当前 `pendingStorytellerAction` 的可用状态、prompt、输入类型、目标数和可选项数量。
- `处理当前` 会打开动态行动表单，由 JS Core 结算。

验收：

- Storyteller 队列不再藏在资料抽屉底部。已完成。
- 玩家能明确看到还有哪些系统动作未结算。第一版完成。

### 3. 剧本手册正式化

现状：

- 已完成独立正式手册第一版。

已落地：

- 独立手册面板。
- 支持全部、镇民、外来者、爪牙、恶魔过滤。
- 左侧角色列表，右侧角色详情。
- 底部展示首夜/其后夜晚顺序摘要和类别统计。
- 主视角身份、魔典标记身份有文本提示。

验收：

- 不需要回 Electron 版查角色能力。第一版完成。
- 手册仍只消费 `scriptHandbook`。已保持。

## P2：视觉和产品质感

### 1. 主菜单和设置页

建议：

- 新局、继续、设置、剧本选择做成正式主菜单。
- 设置先只做显示层：窗口、音量、动画速度占位。

验收：

- Demo 有入口和退出路径。
- 不要求完整存档系统立即完成。

### 2. 音效和过场节奏

建议：

- 私聊发送、投票举手、进入夜晚、进入白天加入轻量音效。
- 只做 UI 反馈，不影响规则时序。

验收：

- 音效可静音。
- 不阻塞 JS Core 结算。

### 3. 响应式布局与拥挤检查

建议：

- 固定检查 16:9、较小窗口、中文长文本。
- 每个浮层都要有最大行数和溢出策略。

验收：

- 不再出现底部托盘文本和按钮互相挤压。
- 长中文句子不会盖住其他控件。

## 暂缓但需要记录：AI 与规则边界

这些不是下一轮最优先目标，但必须进入 backlog，避免之后只靠“发现一个漏洞补一个”。

### 1. AI 对话质量

问题：

- 私聊/公聊回复可能偏模板化。
- AI 可能没有足够自然地表现隐瞒、怀疑、试探、误导。

后续方向：

- 建立 AI 话术评估样例集。
- 区分公开发言、私聊追问、被提名辩解、投票理由。
- 让 AI 输出既符合角色信息边界，又更像玩家。

暂不优先原因：

- 当前更需要先把 Unity 交互面做顺，避免 AI 文本好但 UI 承载差。

### 2. 规则边界与权限审计

已发现并修复的代表问题：

- 非恶魔主视角不应看到恶魔伪装身份。

后续方向：

- 建立“主视角知道什么”的权限矩阵：
  - 普通好人。
  - 恶魔。
  - 中毒/醉酒。
  - 全知魔典。
  - 死亡但有鬼票。
- 审计 Unity viewmodel 是否只导出当前视角可见信息。
- 对 private log、timeline、AI recap、markedRole 做泄露测试。

暂不优先原因：

- 已有基础权限契约兜底。
- 下一步 UI 收束不会新增真相数据来源。

### 3. AI 决策与复盘可信度

问题：

- AI suspicion / vote / nomination 逻辑还需要更系统地验证。
- AI 复盘摘要目前只是首层 trail，不是完整证据簿。

后续方向：

- 做可钻取 evidence 面板。
- 给 AI 投票和提名加可解释理由。
- 加 replay 对照测试：同一局状态下 AI 输出稳定且不泄露信息。

暂不优先原因：

- 复盘数据已经有 `aiRecapDetails[]` 主干。
- 当前更应先解决展示层和操作入口。

## 每轮完成标准

每次迁移或 UI 重做至少满足：

- 不改变 JS Core 规则归属。
- 必要时补 `unity_viewmodel_contracts` 或 `unity_action_bridge_contracts`。
- 运行：

```powershell
node --check scripts\unity_viewmodel.js
node --check scripts\unity_action_bridge.mjs
npm test
```

- Unity build 成功。
- Unity exe smoke 通过。
- 更新 `UNITY_MIGRATION_MATRIX.md` 或本 roadmap。
