# AI Agent 完成度审计（2026-05-13）

本轮审计关注局内 AI agent 的“能不能像玩家一样聊天、思考、行动”，不评价 Unity UI。

## 当前完成度

| 模块 | 状态 | 完成度 | 说明 |
| --- | --- | --- | --- |
| 私聊回答 | 可用，持续打磨 | 75% | 能识别身份、夜间信息、怀疑、信任、投票、计划等问题；会引用 agent 可见证据；会承接上一轮焦点；`questionToAsk` 已接入私聊追问。仍需要更多问法覆盖，尤其是玩家含糊追问和反问。 |
| 公聊发言 | 可用，持续打磨 | 70% | 有公开跳身份、范围声明、压力发言、提名前压力、投票倾向；`questionToAsk` 已接入公聊追问；发言预算收束到短句。仍偏事件驱动，缺少真正多人来回插话的节奏。 |
| AI-AI 私聊 | 基本可用 | 65% | AI 间可交换弱线索，不向第三方公开原文；邪恶方可以私下协调。下一步需要把“谁和谁私聊过”更自然地变成公聊弱线索，而不是显式信息。 |
| 主动私聊 | 基本可用 | 65% | AI 可主动找玩家，玩家可接受/拒绝；会给出原因。仍需要 Unity 侧把突发私聊表现得更轻，不打断主流程。 |
| 思考框架 ThoughtFrame | 已接入核心入口 | 70% | `buildAIThoughtFrame` 会生成关注目标、公开/私聊追问、身份暴露需求、提名准备度；已用于公聊、私聊、提名部分流程。它是轻量“意图层”，不是全局最优解。 |
| 证据/视角隔离 | 较稳 | 80% | agent view、evidence contract、statement memory、KG pressure 已覆盖主要对话入口；好人不会因为话术看到恶魔 bluff/邪恶互认。仍有少量历史函数直接读裸 state，需要继续迁移。 |
| 决策能力 | 可玩级 | 65% | 能根据怀疑、证据、KG pressure、人格、票型做提名/投票倾向；不是纯随机。仍缺更自然的“社交风险”和“局势窗口”判断。 |
| 人格和话术 | 初步成型 | 60% | 有 speech budget、语境化表达、差分输出、邪恶方表演语料；重复回答会做可见差分。仍不是 LLM 级自然语言，需要继续扩充 act 到人话渲染器。 |

## 本轮新增/收束

- `questionToAsk` 已接入私聊追问：AI 回答怀疑、计划、投票时会给出下一句想追的问题。
- `questionToAsk` 已接入公聊追问：公聊压力/观察发言会把追问压进短句里，避免额外第三句。
- 私聊追问会和当前 focus 对齐，避免问 A 时突然追问玩家本人。
- 重复发言会进入差分渲染，减少连续两次完全相同的回答。
- 对话 smoke 从 14 个 warning 降到 0 个 warning。

## 主要剩余问题

- 公聊还不是真正的多人自由对话，更像 conversation clock 驱动的轮流短发言。
- 私聊对复杂自然问法仍有误判风险，例如玩家一句话里同时问身份、信任、投票时，优先级还需要更像人。
- ThoughtFrame 目前影响“关注、追问、暴露、提名准备度”，还没有形成完整的长期计划。
- 邪恶方能伪装和嫁祸，但还缺“主动编世界观并维护”的连续策略。
- 语言自然度已经比模板强，但仍依赖本地规则和语料，不能达到在线 LLM 的开放式聊天能力。

## 下一步建议

1. 做“问答路由器”：先把玩家问题拆成一个主问题和一个副问题，再决定是否先答、追问、拒答或要求补充。
2. 做“公聊插话器”：让 AI 根据上一句发言选择跟进、反驳、附议、转移目标，而不是只按轮次生成。
3. 做“邪恶方世界观维护”：邪恶 AI 记录自己构造过的假链条，后续公私聊都围绕它继续演。
4. 做“局势窗口”：把剩余人数、死亡、提名预算、票数、上台风险纳入提名/投票决策。
5. 扩充 act 渲染器：把内部动作统一转成短、口语、可差分的人话，再进入公聊/私聊/互辩。
