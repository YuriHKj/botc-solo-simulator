## 样本概况

- 剧本：暗流涌动（Trouble Brewing）
- 阶段：第 1 天，私聊阶段
- 被问对象：7号
- Debug truth：7号为占卜师
- 玩家连续提问：
  - “你愿意先给个大概身份吗？”
  - “你有什么别人能对得上的信息吗？”
  - “你昨晚得到了什么信息？”

## 发现的问题

1. 问答意图匹配不稳定

   “有什么别人能对得上的信息吗”没有稳定进入夜间信息问答路径，而是可能落到通用怀疑/追问路径，导致 AI 回答“我先看 8号，接下来问身份和昨晚信息”。这不是在回答玩家问题。

2. 信息位缺少 BOTC 格式推理

   7号是占卜师，但回答一直停在“查验类信息位，昨晚看过两个人”的泛化范围。BOTC 里大部分信息位都有可识别格式，占卜师自然表达应逐步接近“每晚看两个人，得到是/否结果”，连续追问时应能给“查验 X 与 Y，结果：是/否”。

3. 披露策略只有“范围”和“硬跳”两档

   D1 信息位不一定要第一句硬跳身份，但当前缺少中间层：可以说明信息格式、可验证方式、是否暂缓公开目标。缺少这层会让 AI 显得逃避，而不是像玩家在权衡。

4. 公聊公开跳身份偏弱

   现有公聊有软范围披露，但有时整轮仍以点名追问为主，缺少自然的“我公开跳身份/我先报 X”的台面信息锚点。这样玩家很难从公聊建立身份格局。

5. 局部话术模板感

   “结果先别急着公开摊完”等表达重复出现时，像规则模板，不像玩家临场说话。

## 本轮修复

- 扩展私聊意图识别：
  - “对得上 / 能对上 / 可验证 / 能验证 / 硬信息 / 查谁 / 看谁 / 目标 / 结果”等词会进入夜间信息线。
  - 明确身份问题仍进入身份线，避免把“你是什么身份”误路由为夜间信息。

- 增加角色信息格式层：
  - 占卜师：每晚看两个人，得到一个“是/否”结果。
  - 洗衣妇/图书管理员/调查员：两个人里有一个对应某个身份或类型。
  - 共情者：左右两侧存活邻居里有几个邪恶。
  - 厨师：场上有几对相邻邪恶玩家。
  - 送葬者：看当天被处决玩家身份。
  - 其他已预留梦卜者、博学者、钟表匠、守鸦人格式。

- 调整连续追问升级：
  - 第一次问身份范围：可以范围跳。
  - 问“别人能对得上的信息”：给角色信息格式。
  - 再问“昨晚得到了什么信息”：升级为 BOTC 格式结果，例如“查验 4号 与 6号，结果：否”。

- 修正范围话术：
  - 首次范围披露从“刚才已经给过范围”改为“我先给范围”。
  - 占卜师范围句改成“目标和结果先留一点空间”，减少模板感。

- 公聊公开跳身份策略：
  - 第 1 天首轮公聊如果还没有公开 claim，会强制一个合适 AI 做可见硬跳。
  - 仍保留防爆机制：不会让全部 AI 在首轮集体硬跳。
